# 07 Formatting revision for Lecture 1-3 notes

この改訂では、ユーザーのレビュー指摘に基づき、Lecture 1-3 の出力方式を変更した。

## 出力単位

- Lecture 1, Lecture 2, Lecture 3 はそれぞれ別PDFとして出力する。
- 結合PDFはレビュー用の副産物とし、標準出力では各回別PDFを優先する。

## ページ設計

- 原則として、1スライドを1ノート単位とする。
- 上部に元スライドPDFの該当ページを直接埋め込む。
- 下部は2段組みを基本とする。
- 内容が多い場合、過去問関連や確認問題は次ページへ流れてよい。

## 項目構造

各ノートページは以下の順序で構造化する。

1. 日本語訳
2. 解説
3. 試験対策ポイント
4. 過去問関連
5. 確認問題と解答

ただし、表紙、目次、事務情報など、理論的学習対象でないページでは、解説・過去問・確認問題を無理に追加しない。

## 日本語訳

- 日本語訳欄では、解説や要約を混ぜない。
- スライドの箇条書き構造を可能な限り維持する。
- 階層構造がある場合、LaTeXでもネストした itemize として表現する。

## 解説

- スライドに出る概念を、初学者が理解できるように説明する。
- 試験で「説明せよ」と問われた際に、そのまま答案の骨格として使える水準を目標にする。
- 例、用語の定義、なぜ重要か、モデル内での役割を含める。

## 過去問関連

- 年度、設問番号、配点を明記する。
- 問題内容は、過去問本文の丸写しではなく、問われている内容を構造化して要約する。
- 期待される回答を必ず書く。
- 複数の過去問が関係する場合は、すべて構造化して列挙する。

## 確認問題

- 重要スライドでは複数の確認問題を置く。
- 解答は、試験答案に使える水準よりもやや詳しく書く。
- 単なる暗記ではなく、別事例へ適用できる理解を促す。
