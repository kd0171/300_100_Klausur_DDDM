       “Standard” Static Vehicle Routing Problem
       • Depot, customers, vehicles
       • Constraint: capacity
       • Objective: minimize travel time

       • Decisions
         • Assignment of customers to vehicles
         • Sequence of customers per vehicle

       • Modeling: mixed-integer programming
       • Solution: set of routes

       • No consideration of temporal developments
       • No consideration of stochasticity

       Decision Point
       • Decision points define when a decision is made.
       • Notation: 𝑘 = 0, … , 𝐾
       • Not necessarily equidistant and known

       • Examples:
         • Every day, every hour, etc.
         • Whenever a vehicle served a customer/ a job is produced
         • Whenever a new customer requests/ new job comes in
         • Self-Induced (“I will wait 5 minutes and then decide”)

       • The time a decision is made depends on its “importance” to the stakeholders
       • Tradeoff between fast reaction and information gain.

       Decisions and Rewards
       • Decisions are made about the use/spending of resources to serve demand
       • Notation: 𝑥𝑘 ∈ 𝑋(𝑆𝑘 ) in state 𝑆𝑘

       • Feasibility depend on
         • Constraints in resource use (travel/set-up times, capacities, etc.)
         • Constraints in service of demand (deadlines, amount, etc.)

       • Each decisions causes costs and/or rewards 𝑅 𝑆𝑘 , 𝑥𝑘 :
         • Costs in use of resources
         • Rewards in service of demand

       • Often, either costs or rewards are moved to the constraints.
       • Reward can be a random variable

                                                                                                             Action:
       • Decisions and rewards can be modeled
         either as immediate action or as plan updates.
       • Immediate action is the least commitment
         necessary (next customer to serve, next job
         to produce)
       • Plan updates consider a sequence of tentative
         actions where only the first is implemented
       • Each model type requires different reward                                                           Plan:

       • Opinion: Plan updates are more intuitive and
         connect better to methodology

       Post-Decision State
       • Combination of state 𝑆𝑘 and decision 𝑥𝑘 leads to post-decision state 𝑆𝑘𝑥
       • Deterministic part of the overall transition from state to state.
       • Often contains similar information as the state with changes (e.g., no open requests/jobs)

       • Can be seen as the “current plan.”

       • Explicit modeling supports
         • Communication (We see the implementation of the decision)
         • Methodology (compare Bellman Equation, max 𝑅(𝑆𝑘 , 𝑥) + 𝑉(𝑆𝑘𝑥 ))

       Stochastic Information and Transitions
       • Observation of stochastic information 𝜔𝑘 given post-decision state 𝑆𝑘𝑥 leads
         to new state 𝑆𝑘+1 .
       • Stochastic part of the overall transition from state to state.
       • Each stochastic information has a probability (in theory).
       • If transition is not self-evident, sometimes transition function is explicitly modeled
         (compare Powell 2011)

       • Transitions comprise:
         • Observation of new customers/jobs/demand
         • Customer selecting products
         • Etc.

       Coding MDPs
       For coding MDPs, five components are required:

       1. Exogeneous information, for example, lists of jobs/customers with all relevant details
          (duration, location, potentially choice model details, etc.)
       2. Simulation loop, for example, over time or the job/customer list
       3. State information as combination of previous post-decision state and new information
       4. Decision policy, assigning a decision to a state (see next week) and creating reward and
          post-decision state
       5. Variable to track observed rewards
