       Sampling + Mixed Integer Programming
       • Idea: Instead of state transitions, sample stream of stochastic information (for example,
         customer requests, production orders, demand, etc.)

       •      Extend state information with sampled information
       •      Solve mixed-integer program (maximize reward)
       •      Remove parts of solution that consider sampled information
       •      Implement (partial) solution

       • Advantage: Conventional MIP-solvers can be used
       • Disadvantage: Only one sample used for decision making (overfitting)

       Multiple Scenario Approach (MSA)
       • Bent, Van Hentenryck 2004
       • Two-stage stochastic programming “light”
       • Sample set of scenarios, determine solution that fits all scenarios best

       •      Sample several (streams of) information model realizations (e.g., sets of requests)
       •      Solve deterministic MIPs individually
       •      Clean solutions from augmented information (and repair, if necessary)
       •      Compare solutions and select “most similar” solution

       • Advantage: Conventional MIP-solver can be used
       • Disadvantage: stochasticity only captured implicitly in solution selection, no dynamism

       MSA: Consensus Function
       • The consensus function determines the “most similar” solution
       • Nice idea, but highly problem dependent
       • Consensus function for production planning:
         • Solution: Sequence of jobs per production line
         • Consider only next job each line produces
         • Similarity measure between two solutions: number of identical jobs
         • Select solution with maximum similarity
       • Example: 3 lines, A, B, C, six jobs 1,2,3,4,5,6
         • Solution 1:      A12, B34, C56
         • Solution 2:      A13, B45, C26
         • Solution 3:      A21, B35, C46
       • Solution 1 is the most representative solution and (partially) implemented

       In each state
       • Generate a set of scenarios

       • Scenario is a known ranking of customers
              for each supplier

       • Solve individual scenarios with some IP-magic

       • Derive “consensus” solution from scenario solutions

       • Observe customer selection of supplier

       • Perform final assignment of customers to supplier

       •      Preferences of suppliers with regard to attributes of the current state are not known
       •      Distance to the customer and travel time are identified as meaningful attributes
       •      The weights of attributes are modified for each supplier at random
       •      Solving the respective MIP leads to a ranking/rejection of requests for suppliers

       Look-ahead: Advantages and Disadvantages
       • Advantages:
         • Full state information is considered
         • Stochasticity is considered explicitly
         • Full decision space XOR
         • Dynamic developments can be considered

       • Disadvantages:
         • Runtime
         • Information model: only a few realizations can be considered
         • Decision model: only a few decisions can be considered in simulation
         • Divergence from realistic developments with increasing horizon.
         • Conventional solvers may not be suitable, e.g., for rollout algorithms
